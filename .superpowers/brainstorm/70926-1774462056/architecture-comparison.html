Current vs. Proposed Architecture — Visualization

How should we show the shift from the funnel model to the Orbital Gravity Well? Each approach tells the same story differently.

A

Side-by-Side Diagrams

Two SVG diagrams rendered next to each other. Left: the traditional funnel (wide top, narrow bottom, annotated with drop-off points). Right: the Orbital Gravity Well (concentric rings, each representing a depth level, with arrows showing audience moving inward). Labels explain each layer. The contrast is immediate — one shape narrows and loses people, the other deepens and retains them.

Best for: Visual learners who grasp structure from shape. The funnel silhouette is already culturally understood — the contrast lands fast.
B

Annotated Flow Comparison (Expanded FlowPulse)

An expanded version of the FlowPulse component already built — two parallel flows, but with much richer annotation. Each stage is labeled with: what it is in the current model, what it becomes in the new model, and why the change matters. The current model's drop-off is shown in Pink as a loss signal. The proposed model's retention is shown in Teal expanding outward.

Best for: Showing the process difference, not just the shape difference. Reuses established visual language from the homepage — the visitor has already seen this format and trusts it.
C

Stacked Scaffold — Before / After Layers

Two sets of Scaffold bands stacked vertically. The first set (Teal borders, labelled "CURRENT") shows Dan Koe's three existing funnel stages with an honest description of what's working and what's leaking. A thin Pink divider line reads "THE SHIFT". Below it, a second set (Orange borders, labelled "PROPOSED") shows the five Orbital Gravity Well phases — each band expanding on hover to reveal the learning science principle behind it.

Best for: Demonstrating depth of thinking. Each band is a paragraph, not a label. The visitor reads their way through the analysis. Fully reuses the Scaffold component already built.
NOTE ON SEQUENCING

These options are not mutually exclusive. Options A or B work well for Act 2 (Current State diagnosis — fast, visual, visceral). Option C works well for Act 3 (Our Architecture — deep, annotated, methodical). The most powerful approach may be: A or B first to diagnose, then C to prescribe.
